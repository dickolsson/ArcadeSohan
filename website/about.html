⚙️ À Propos

Comment fonctionne la console Arcade Sohan

🎮 Le Projet

Arcade Sohan est une console de jeux faite maison construite avec un Arduino Uno, un écran OLED, un joystick et un buzzer. Ce projet éducatif enseigne l'électronique et la programmation à travers la création de jeux vidéo rétro!

Conçu pour un étudiant de 10 ans, le projet combine des concepts simples avec des résultats impressionnants. Chaque module est documenté en français et anglais pour faciliter l'apprentissage.

🔌 Composants Hardware

🖥️

Arduino Uno

Microcontrôleur principal

  • 32 KB Flash (code)
  • 2 KB RAM (variables)
  • 16 MHz
  • Programmé en C++
📺

Écran OLED I2C

Affichage 0.96"

  • 128×64 pixels
  • Monochrome (blanc)
  • Interface I2C (2 fils)
  • Pins: SDA (A4), SCL (A5)
🕹️

Joystick Module

Contrôle 2 axes + bouton

  • Axe X: Pin A0 (0-1023)
  • Axe Y: Pin A1 (0-1023)
  • Bouton: Pin D7
  • Centre ~512 pour X et Y
🔊

Buzzer Passif

Sons et mélodies

  • Pin D8
  • Nécessite signal PWM
  • Fonction tone()
  • Fréquences 100-5000Hz
🔋

Alimentation

USB ou batterie

  • 5V via USB
  • ou batterie 9V
  • Consommation ~100mA
  • Portable!
🔌

Breadboard

Prototype sans soudure

  • 830 points de connexion
  • Rails d'alimentation
  • Facile à modifier
  • Idéal pour apprendre

🔧 Configuration des Pins

Composant Pin Arduino Type Description
OLED SDA A4 I2C Data Données I2C pour l'écran
OLED SCL A5 I2C Clock Horloge I2C pour l'écran
Joystick X A0 Analog Input Mouvement horizontal (0-1023)
Joystick Y A1 Analog Input Mouvement vertical (0-1023)
Joystick Button D7 Digital Input Bouton d'action (pull-up)
Buzzer D8 Digital Output Signal PWM pour le son

💡 Note: L'écran OLED fonctionne à 3.3V mais a un régulateur intégré, il peut donc être alimenté en 5V depuis l'Arduino.

🎯 Design Patterns Utilisés

🏭 Object Pool

Gestion d'ennemis et projectiles sans allocation dynamique. Maximum 6 objets par pool.

🎮 Game State Machine

États du jeu: EN_COURS, PAUSE, TERMINE, QUITTER. Transitions claires entre états.

📦 Module Pattern

Chaque fichier .h est un module autonome avec son propre namespace (préfixes mh_, av_, etc.).

🎲 Procedural Generation

Même seed = même résultat. Niveaux infinis sans stocker de données.

💾 Optimisation Mémoire

⚠️ Le Défi

L'Arduino Uno a des ressources très limitées:

  • 32 KB Flash - pour le code
  • 2 KB RAM - pour les variables
  • Buffer OLED = ~128 bytes
  • Stack = ~200-400 bytes
  • RAM disponible: ~1200 bytes!

✅ Solutions Appliquées

  • PROGMEM - Stocker niveaux/textes en Flash 
    NIVEAU_PROGMEM(niv1, {0,56,40});
  • int8_t - 1 byte au lieu de 2 
    int8_t x, y;  // au lieu de int
  • Pools limités - Max 3-6 objets 
    ObjetMobile balles[4];  // pas 100!
  • Page Buffer - OLED en 8 strips 
    // 128 bytes au lieu de 1024
  • Inline Functions - Physics.h sans RAM 
    inline bool phys_touchePoint(...)

📊 Résultats

Flash utilisée ~28 KB / 32 KB
RAM utilisée ~1100 bytes / 2048 bytes

✨ 3 jeux complets dans 2KB de RAM!
