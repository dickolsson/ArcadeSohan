🔧 Systèmes Communs

Les modules partagés par tous les jeux de la console

📦 Modules Réutilisables

Pour éviter de dupliquer du code et économiser de la mémoire, tous les jeux partagent des modules communs. Chaque module est un fichier .h qui peut être inclus dans n'importe quel jeu.

📺 Display.h - Affichage

Module pour dessiner sur l'écran OLED. Utilise le mode "page buffer" pour économiser la RAM (128 bytes au lieu de 1024).

Fonctions principales :

  • DESSINER_ECRAN { } - Macro obligatoire pour dessiner
  • ecrireTexte(x, y, "text", size) - Afficher du texte
  • dessinerRectangle(x, y, w, h) - Dessiner des formes
  • dessinerCercle(x, y, r) - Dessiner un cercle
DESSINER_ECRAN {
  ecrireTexte(0, 0, "Score:", 1);
  ecrireNombre(40, 0, score, 1);
}

🕹️ Input.h - Contrôles

Module pour lire le joystick et le bouton. Gère la détection des directions et les pressions de bouton avec debouncing.

Fonctions principales :

  • lireJoystick() - Mettre à jour l'état (appeler chaque frame)
  • joystickHaut() - Vérifier direction haut
  • joystickBas() - Vérifier direction bas
  • joystickGauche() - Vérifier direction gauche
  • joystickDroite() - Vérifier direction droite
  • boutonJustePresse() - Détection de clic (edge detection)
lireJoystick();
if (joystickGauche()) joueurX -= 2;
if (boutonJustePresse()) tirer();

🎵 Melodies.h - Sons

Mélodies pré-faites pour différentes actions du jeu. Utilise le buzzer passif pour créer des sons rétro.

Mélodies disponibles :

  • melodieStartup() - Démarrage de la console
  • melodieTir() - Son de tir
  • melodieGameOver() - Fin de partie
  • melodieAlerteBoss() - Boss apparaît
  • melodieVictoireBoss() - Boss vaincu
  • melodieNiveauSup() - Level up
if (toucheEnnemi) {
  melodieMonstreTouche();
  score += 25;
}

🔬 Physics.h - Physique

Fonctions pour la détection de collision et le mouvement. Zéro coût en RAM car toutes les fonctions sont inline.

Fonctions principales :

  • phys_touchePoint(x1, y1, x2, y2, dist) - Collision cercle
  • phys_toucheBoite(x, y, w, h, ...) - Collision rectangle
  • phys_bougerVers(&x, &y, tx, ty, speed) - IA de poursuite
  • phys_clampEcran(&x, &y, margin, topMargin) - Rester à l'écran
// Collision joueur-monstre
if (phys_touchePoint(joueurX, joueurY, 
                     monstreX, monstreY, 10)) {
  gameOver();
}

📦 Objects.h - Objets

Système de pool pour gérer plusieurs objets (ennemis, projectiles) sans allocation dynamique de mémoire.

Structures et fonctions :

  • ObjetSimple - Objets statiques (collectibles, checkpoints)
  • ObjetMobile - Objets avec vélocité (balles, ennemis mobiles)
  • obj_creer(pool, size, x, y, type) - Créer un objet
  • obj_touchePoint(pool, size, x, y, dist) - Collision
  • Maximum 6 objets par pool
ObjetMobile balles[4];
objm_initialiser(balles, 4);
objm_creer(balles, 4, x, y, vx, vy, 1);

🎲 Procedural.h - Génération

Génération procédurale de niveaux et positions. Même seed = même résultat pour des niveaux rejouables et testables.

Fonctions principales :

  • proc_genererPosition(seed, idx, &x, &y, margin) - Position aléatoire
  • proc_genererPlateformes(level, plat, n, diff) - Créer des plateformes
  • proc_calculerDifficulte(level) - Retourne difficulté 1-3
  • proc_genererLoinDe(seed, idx, &x, &y, ...) - Loin d'un point
// Niveau 1 = toujours les mêmes plateformes
proc_genererPlateformes(niveau, plat, 5, 1);

💾 ProgMem.h - Mémoire

Stockage de données dans la Flash (32KB) au lieu de la RAM (2KB). Essentiel pour tenir dans l'Arduino Uno.

Macros et fonctions :

  • NIVEAU_PROGMEM(name, {...}) - Stocker un niveau
  • TEXTE_PROGMEM(name, "text") - Stocker du texte
  • pm_charger3Colonnes(src, dest, n) - Charger en RAM
  • pm_lireTexte(txt) - Lire du texte
  • Économise des centaines de bytes de RAM
NIVEAU_PROGMEM(niv1, {0,56,40, 45,46,30});
pm_charger3Colonnes(niv1, plat, 2);

👤 Personnages.h - Avatars

Système de personnages débloquables. Les joueurs gagnent des étoiles pour débloquer de nouveaux avatars avec différents designs.

Personnages et fonctions :

  • 4 personnages: Blob (gratuit), Bonhomme, Héros, Champion
  • Système d'étoiles: 0, 99, 300, 600
  • pers_dessinerPlateforme(x, y, perso, dir, frame) - Vue côté
  • pers_dessinerVueHaut(x, y, taille, perso, dir) - Vue dessus
  • ajouterEtoiles(nombre) - Gagner des étoiles
pers_dessinerVueHaut(joueurX, joueurY, 8,
                     personnageActuel, direction);

✨ Avantages des Modules

♻️

Réutilisabilité

Écrire le code une fois, l'utiliser partout. Économise du temps de développement.

💾

Économie de Mémoire

Partager le code entre jeux évite la duplication. Un seul Physics.h pour tous!

🐛

Maintenance Facile

Corriger un bug dans un module = corriger pour tous les jeux en même temps.

📚

Code Organisé

Structure claire : chaque module a une responsabilité unique et bien définie.

⚡

Développement Rapide

Créer un nouveau jeu est plus rapide avec tous les outils déjà disponibles.

🎓

Facile à Apprendre

Chaque module peut être étudié indépendamment. Apprentissage progressif.
