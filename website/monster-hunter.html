Monster Hunter

Action Vue du dessus Tir

📝 Description

Monster Hunter est un jeu d'action en vue du dessus où tu dois chasser des monstres, manger de la nourriture pour recharger tes munitions, et affronter des boss épiques! Plus tu avances, plus les monstres deviennent rapides et dangereux.

🎮 Contrôles

  • Joystick ↑↓←→ : Déplacer le personnage
  • Bouton : Tirer (coûte 1 munition)
  • Objectif : Mange la nourriture pour recharger tes munitions

✨ Fonctionnalités

  • 🎯 Système de munitions : Gère tes tirs intelligemment
  • 👹 Boss tous les 5 niveaux : Défis épiques avec plus de points de vie
  • ⭐ Système d'étoiles : Gagne des étoiles pour débloquer des personnages
  • 🎲 Génération procédurale : Chaque partie est différente
  • 🔊 Effets sonores : Sons de tir, de coup, et d'alerte boss

💻 Comment ça marche?

🗂️ Structure du Code

Le jeu utilise plusieurs modules pour organiser le code:

  • MonsterHunter.h - Logique principale du jeu
  • Objects.h - Gestion des objets (monstres, nourriture)
  • Procedural.h - Génération aléatoire des positions
  • Physics.h - Détection de collision
  • Personnages.h - Système de personnages

🎯 Système de Collision

Le jeu utilise la fonction phys_touchePoint() pour détecter:

  • Quand le joueur touche un monstre (Game Over)
  • Quand une balle touche un monstre (Tué!)
  • Quand le joueur mange la nourriture (Recharge)
// Exemple de code (Example code)
if (phys_touchePoint(joueurX, joueurY, 
                     monstreX, monstreY, seuil)) {
  // Collision détectée!
}

🎲 Boss System

Chaque 5 niveaux, un boss apparaît avec plus de vie:

// Boss tous les 5 niveaux (Boss every 5 levels)
if (niveau % 5 == 0) {
  estBoss = true;
  vieBoss = 3 + (niveau / 5);
  melodieAlerteBoss();
}

💾 Optimisation Mémoire

Pour tenir dans les 2KB de RAM de l'Arduino:

  • Maximum 3 monstres en même temps
  • Maximum 4 balles en même temps
  • Utilisation de int8_t (1 byte) au lieu de int (2 bytes)
  • Objets réutilisés avec un système de pool
