Aventurier

Plateforme Puzzle Physique

📝 Description

Aventurier est un jeu de plateforme où tu dois sauter de plateforme en plateforme pour atteindre la porte magique! Les premiers niveaux sont faits à la main, puis le jeu génère automatiquement des niveaux infinis de plus en plus difficiles.

🎮 Contrôles

  • Joystick ←→ : Déplacer gauche/droite
  • Bouton : Sauter (seulement si au sol)
  • Objectif : Atteindre la porte dorée ✨

✨ Fonctionnalités

  • 🎯 Physique réaliste : Gravité et vitesse de saut naturelles
  • 🗺️ 4 niveaux faits main : Tutoriel progressif
  • ∞ Niveaux infinis : Génération procédurale après le niveau 4
  • ⚡ Difficulté croissante : Sauts de plus en plus difficiles
  • 🚪 Porte animée : Effet brillant pour la sortie

💻 Comment ça marche?

🎮 Système de Saut

Le jeu utilise une vraie physique de gravité:

// Appliquer la gravité (Apply gravity)
vitesseY += GRAVITE;  // Tombe de plus en plus vite

// Sauter si au sol (Jump if on ground)
if (boutonJustePresse() && auSol) {
  vitesseY = -FORCE_SAUT;
  auSol = false;
}

🗺️ Niveaux Mixtes

Les 4 premiers niveaux sont stockés en mémoire Flash (PROGMEM), puis le jeu passe à la génération procédurale:

// Créer un niveau (Create a level)
void creerNiveau() {
  if (niveau <= 4) {
    // Charger niveau fait main (Load handmade)
    pm_charger3Colonnes(niveaux[niveau], plat, 5);
    pm_chargerPaire(portes[niveau], &porteX, &porteY);
  } else {
    // Générer procédural (Generate procedural)
    int diff = proc_calculerDifficulte(niveau);
    proc_genererPlateformes(niveau, plat, 5, diff);
    proc_genererPorte(plat, 5, &porteX, &porteY);
  }
}

🔍 Détection de Collision

Le jeu vérifie si le joueur touche une plateforme:

// Vérifier collision plateforme
auSol = false;
for (int i = 0; i < 5; i++) {
  if (phys_toucheBoite(joueurX, joueurY, 6, 8,
                       plat[i][0], plat[i][1], 
                       plat[i][2], 4)) {
    auSol = true;
    vitesseY = 0;
    break;
  }
}

💾 Économie de Mémoire

Astuces pour économiser de la RAM:

  • Seulement 5 plateformes maximum par niveau
  • Niveaux faits main stockés en Flash, pas en RAM
  • Une seule fonction creerNiveau() au lieu de 10+ fonctions
  • Variables int8_t pour les petites valeurs
